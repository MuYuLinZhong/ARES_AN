# NAC1080 无源锁安卓应用详细设计方案 (V2.0)

## 1. 账户与权限管理 (Account & Permissions)

### 1.1 登录页 (Login)
*   **登录方式**：手机号 + 密码登录。
*   **密码可见性切换**：密码输入框右侧提供"显示/隐藏密码"按钮（EyeOff 图标）。
*   **忘记密码**：提供"Forgot Password?"入口链接。
*   **注册入口**：底部提供"Sign Up"跳转入口，供新用户注册。
*   **登录成功**：跳转至首页（Home）。

### 1.2 修改密码页 (Update Password)
*   **三段式输入**：当前密码 → 新密码 → 确认新密码，三个独立输入框，均支持密码可见性切换。
*   **安全提示**：页面顶部展示密码强度建议文案（至少 8 位随机字符）。
*   **提交**：点击"Update Password"按钮后返回设置页。

### 1.3 设备权限管理
*   **添加授权用户**：Owner 在设备详情页通过手机号邀请其他用户，被邀请用户将收到 SMS 邀请。
*   **撤销授权**：Owner 可对已授权用户点击删除图标，触发二次确认弹窗后撤销权限，操作不可逆，被撤销用户立即失去操作权限。
*   **权限角色展示**：授权用户列表中区分 Owner（标注 Default 标签）和普通授权用户。

## 2. 核心交互流程：先指令，后校验 (Auth-After-Command)
本流程适配 NAC1080 无源特性，采用"意图先行、硬件出题、云端解题、硬件验证"的逻辑。

### 2.1 交互时序图逻辑
1.  **用户触发**：在 App 首页点击"Unlock（开锁）"或"Lock（关锁）"按钮。
2.  **发送操作位 (Intention Bit)**：
    *   App 立即通过 NFC 向 NAC1080 发送操作指令位（例如：`0x01` 代表开锁，`0x02` 代表关锁）。
3.  **硬件响应随机数 (Challenge)**：
    *   NAC1080 接收到操作位后，内部生成一个**随机数 (Random Number)**。
    *   硬件将该随机数通过 NFC 回传给 App。
4.  **云端加密与校验 (Cloud Security)**：
    *   App 将"设备 ID + 随机数 + 操作位"发送至云端。
    *   **云端动作**：
        *   **权限校验**：检查该用户是否有权操作此设备。
        *   **实时日志**：云端在接收到请求的瞬间，**立即记录一条操作请求日志**（包含时间、用户、设备、操作类型）。
        *   **加密计算**：云端使用该设备的专属密钥对随机数进行加密，生成**认证密文**。
    *   云端将密文下发至 App。
5.  **硬件比对与执行 (Execution)**：
    *   App 将密文发送给 NAC1080。
    *   **锁具内部验证**：NAC1080 比对云端密文与自身计算结果。
    *   **执行指令**：
        *   若比对通过：执行对应的开锁或关锁机械动作。
        *   **关锁特有逻辑**：执行关锁后，硬件自动进行机械机构检测，确认门栓是否到位。
6.  **结果反馈**：
    *   硬件返回最终执行状态（成功/失败原因）。
    *   App 更新 UI，并向云端同步最终执行结果（补全日志状态）。

### 2.2 操作进度 UI（底部弹出面板）
*   点击 Unlock/Lock 后，屏幕底部弹出半屏操作面板（Bottom Sheet），覆盖背景模糊处理。
*   面板内展示**环形进度条**（0%→100%），实时反映 NFC 通信与云端校验进度。
*   面板内展示当前步骤文案（如"Unlocking device... / Scanning for NFC signal..."）。
*   提供"Cancel"按钮，允许用户中途取消操作，面板关闭并重置进度。

## 3. 核心功能特性描述 (Functional Highlights)

### 3.1 首页 (Home)
*   **顶部栏**：
    *   左侧：用户头像按钮，点击跳转设置页。
    *   中间：当前激活设备名称（Active Device）。
    *   右侧：通知铃铛按钮（Bell 图标）。
*   **NFC 扫描动效**：页面中央展示 NFC 图标 + 旋转虚线圆环动效，视觉提示用户靠近锁具。
*   **操作按钮区**：底部两个并排按钮——"Unlock（蓝色主色）"和"Lock（深色次要色）"。
*   **操作提示文案**：按钮下方展示"Keep NFC close after clicking"提示。

### 3.2 设备列表页 (Devices)
*   **搜索栏**：支持按设备昵称搜索过滤。
*   **设备卡片**：每张卡片展示设备图标、设备名称、设备 ID，并提供"Unlock（快捷开锁）"和"Manage（进入详情）"两个操作按钮。
*   **新增设备入口**：右上角"+"图标按钮，跳转至添加设备页。

### 3.3 添加设备页 (Add Device)
*   **设备昵称输入**：文本输入框，用于为新设备命名（如"Office"）。
*   **NFC 靠近按钮**：点击"Approach and Add"后进入 NFC 扫描等待状态。
*   **引导动效**：页面下方展示手机靠近锁具的示意图（手机模型 + 锁具模型 + 信号波纹动效）。
*   **操作说明**：文案提示用户将手机背面顶部对准锁具 NFC 标志。

### 3.4 设备详情页 (Device Details)
*   **设备基本信息**：展示设备图标、设备名称、设备 ID、序列号（Serial Number）。
*   **授权管理（Share Access）**：
    *   通过手机号输入框 + "Invite"按钮邀请新用户。
    *   授权用户列表展示所有已授权用户（头像、姓名、手机号）。
    *   Owner 条目标注"Default"标签，不可删除。
    *   普通授权用户右侧提供删除（Trash）图标，点击触发撤销确认弹窗。
*   **撤销确认弹窗**：展示被撤销用户姓名，提供"Revoke Access（红色确认）"和"Cancel"两个按钮。
*   **底部操作栏**：
    *   "Save Changes"按钮（蓝色主色）。
    *   "Remove Device"按钮（红色警告色），用于解绑并删除该设备。

### 3.5 设置页 (Settings)
*   **用户信息展示**：头像、用户名、角色标签（如 Admin Profile）、邮箱。
*   **Profile Management（账户管理）**：
    *   手机号展示（可点击修改）。
    *   修改密码入口（跳转 Update Password 页）。
*   **Working Mode（工作模式）**：
    *   在线模式开关（Online Mode Toggle）：开启时要求联网进行密钥认证，关闭时切换为离线模式。
*   **System Settings（系统设置）**：
    *   震动反馈开关（Vibration Feedback Toggle）。
    *   NFC 灵敏度设置（NFC Sensitivity，可选 High/Medium/Low）。
    *   关于我们（About Us，展示版本号）。
*   **退出登录**：底部红色"Logout"按钮，点击后返回登录页。

### 3.6 底部导航栏 (Bottom Navigation)
*   三个固定 Tab：
    *   **Unlock**（Home 图标）→ 首页
    *   **Devices**（Smartphone 图标）→ 设备列表
    *   **Add**（PlusCircle 图标）→ 添加设备
*   当前激活 Tab 高亮显示主色（蓝色），图标填充。

## 4. 预操作机制 (Pre-operation Bit)
*   不同于传统的先认证再操作，本方案先下发操作位。这允许硬件提前进入对应的物理预备状态，缩短用户感知的等待时间。

## 5. 动态安全体系
*   **硬件出题**：随机数由 NAC1080 硬件随机发生器生成，确保每轮操作的唯一性，彻底防御"重放攻击"。
*   **云端解题**：密钥存储在云端 HSM 或安全数据库中，手机端不存储设备密钥，极大提升了系统的安全性。

## 6. 闭环机械检测 (Mechanical Feedback)
*   **关锁确认**：关锁操作不以"指令发出"为准，而以 NAC1080 物理反馈的"机械结构闭合信号"为准。若门未关严，系统将明确告知用户关锁失败。

## 7. 实时日志与审计 (Real-time Logging)
为了确保金融级可追溯性，日志策略调整如下：

*   **云端主动记录**：
    *   **触发即记录**：只要 App 向云端请求加密，云端即视为操作已发生，并生成一条"处理中"的操作记录。
    *   **防篡改**：日志不再完全依赖手机端上传，防止用户在操作后迅速断网或卸载 App 导致记录丢失。
*   **状态回填**：
    *   当操作完成后，App 会异步回传最终的硬件反馈结果，云端将之前的记录状态由"处理中"更新为"成功"或"失败（附带错误码）"。

### 给开发/测试人员的特别说明：
1.  **对于 Android 开发**：NFC 通信需要保持长连接，从发送操作位到接收密文再回传给锁，中间不可断开。
2.  **对于测试人员**：需重点测试"云端已记录日志但 NFC 中途断开"的异常场景，确保日志状态能正确反映"执行中断"；同时测试"撤销授权后被撤销用户立即无法操作"的实时性。
3.  **对于后端开发**：加密接口必须承载日志记录功能，确保每一笔 Token 请求都有案可查；邀请用户接口需触发 SMS 下发。
